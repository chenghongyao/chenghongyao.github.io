旅行者问题(TSP)

发表于2021-06-15|更新于2021-07-20|算法
|阅读量:

TSP问题（旅行者问题）是指旅行家要旅行若干个城市，要求各个城市经历且仅经历一次然后回到出发城市，并要求所走的路程最短。

本文分析了从起点开始经过若干城市1次后到达终点的最短距离，对于回到起点的问题也可以用类似的方法，只需将终点替换为起点。

问题描述

从起点 SSS 出发，经过若干个城市 iii 一次，最终到达终点 TTT。已知所有地点两两之间的距离：dij、diS、diTd_{ij}、d_{iS}、d_{iT}dij​、diS​、diT​，求从起点 SSS 到终点 TTT 的最短距离。

分析

使用 d(i,V)d(i,V)d(i,V) 表示从城市 iii 经过城市集合 VVV (i∉Vi\notin Vi∈/V) 中的城市1次后到达终点 TTT 的最短距离，当 VVV 是空集时，从城市 iii 到终点的最短距离为 d(i,∅)=diTd(i,\empty)=d_{iT}d(i,∅)=diT​。

为了将问题分解，设选择的下一个城市为 k∈Vk\in Vk∈V，则到达终点 TTT 的距离可以表示为：

dik+d(k,V−k)d_{ik} + d(k,V-k) dik​+d(k,V−k)

其中 V−kV-kV−k 表示将城市 kkk 从城市集合 VVV 中移除。

为了使到终点的距离最短，需要遍历 VVV 中的所有城市，选择总距离最短的城市作为下一个城市

d(i,V)=min⁡k∈V[dik+d(k,V−k)]d(i,V) = \underset{k\in V}{\min}\left[d_{ik}+d(k,V-k)\right] d(i,V)=k∈Vmin​[dik​+d(k,V−k)]

此时问题转为求子问题 d(k,V−k)d(k,V-k)d(k,V−k)，不断地对问题分解，最终使经过的城市集合为空集，此时有d(k,∅)=dkTd(k,\empty)=d_{kT}d(k,∅)=dkT​。下图展示了问题的分解过程。

最后用同样的方法计算从起点 SSS 经过所有城市后到达终点的最短距离 d(S,V)d(S,V)d(S,V)。

实现

假设有 NNN 个城市 0,1,2,⋯ ,N−10,1,2,\cdots,N-10,1,2,⋯,N−1，则城市集合 VVV 有 2N2^{N}2N种可能。使用一个 NNN 位的二进制数字 VVV 表示城市集合，当 VVV 的第 iii 位为 111 时表示 iii 城市存在于这个集合中，V=0V=0V=0表示空集， 1<<i1<<i1<<i 表示只有第 iii 个城市的集合， V^(1<<j)V\text{\textasciicircum} (1<<j)V^(1<<j)表示从城市集合 VVV 中移除城市 jjj （V−jV-jV−j）。

使用 dp[i][V]dp[i][V]dp[i][V] 表示从城市 iii 出发经过 VVV 表示的城市集合到达终点 TTT 的最短距离。初始化空集
dp[i][0]=diTdp[i][0]=d_{iT}dp[i][0]=diT​。对于 VVV可以从 111 到 2N−12^N-12N−1 进行遍历。

一般问题中的城市数目都比较少，因为问题的规模是 2N2^N2N，城市数变多会使问题的规模迅速膨胀。
